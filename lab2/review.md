# Лабораторна робота №2: Створення реляційної бази даних (DDL)

## 1. Мета роботи
Перетворити концептуальну ER-діаграму інтернет-магазину (розроблену в Лабораторній №1) на фізичну схему бази даних PostgreSQL за допомогою SQL-запитів.

## 2. Виконані завдання
- Створено базу даних в середовищі **pgAdmin 4**.
- Написано SQL-скрипт (`schema.sql`) для створення 5 взаємопов'язаних таблиць: `Customer`, `Category`, `Product`, `Order` та `OrderItem`.
- Для кожної таблиці визначено відповідні типи даних (`SERIAL`, `VARCHAR`, `NUMERIC`, `TIMESTAMP`, `INTEGER`).
- Налаштовано зв'язки між таблицями за допомогою зовнішніх ключів (`FOREIGN KEY`) із правильними правилами поведінки при видаленні (`ON DELETE CASCADE` та `ON DELETE RESTRICT`).
- Застосовано обмеження цілісності (Constraints) для захисту даних від помилок: `PRIMARY KEY`, `NOT NULL`, `UNIQUE` (для перевірки унікальності email), `CHECK` (ціна > 0, кількість на складі >= 0) та `DEFAULT` (автоматичне встановлення дати реєстрації та базового статусу замовлення).
- Таблиці успішно наповнено тестовими даними (по 4-5 записів для кожної) за допомогою команд `INSERT INTO`.

## 3. Результати та тестування
Скрипт був успішно виконаний без синтаксичних помилок. Для перевірки коректності завантаження даних було виконано базові запити `SELECT * FROM ...`. 

Результати вибірки (докази успішного створення та наповнення таблиць) прикріплені у цій директорії у вигляді скріншотів з pgAdmin:
* `customers.png` — дані таблиці клієнтів;
* `products.png` — дані таблиці товарів з відповідними категоріями;
* `orders.png` — дані таблиці замовлень.
* `erd-diagram.png` — концептуальна схема БД (для зручності перевірки).

## 4. Висновок
Під час виконання лабораторної роботи було практично закріплено навички роботи з DDL-командами (`CREATE TABLE`) та базовими DML-командами (`INSERT INTO`) в СУБД PostgreSQL. База даних повністю готова до подальшої роботи (написання складних SQL-запитів).
